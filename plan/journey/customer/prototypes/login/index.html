Protótipo — CUSTOMER: Login

UCs cobertos: UC-021 (login + seleção de tenant) · UC-023 (trocar tenant)

Journey: customer/login.md · Stories: M124-S02

Fluxo principal — 1 tenant (Caso A)

  • 00 — Hotsite do estabelecimento EXISTS: M12 — entry: botão "Entrar" (com ?redirect=/{slug}/booking)
  • 01 — Tela de login do cliente GAP: /{tenantSlug}/login — shared/login.html
  • 02 — Completar perfil (sem telefone) GAP: prompt pós-login — UC-021 A3 — one-time
  • 03 — Agendamento — Passo 1 (escolha de serviço) EXISTS: ServiceSelectionStep — destino pós-login: retoma o fluxo de booking

Fluxo alternativo — +1 tenants (Caso B)

  • 00 — Login genérico GAP: /auth/login
  • 01 — Selecionar estabelecimento GAP: /select-tenant — UC-021 Caso B — mostra pontos por tenant
  • → Após seleção: mesmo fluxo do Caso A (phone check → agendamento passo 1)

Estados alternativos (unhappy paths)

  • 01b — Erro de autenticação GAP: /auth/error — no-tenant / tenant-deactivated (compartilhado com staff)

O que este protótipo valida

  • ☐ A tela de seleção de tenant (Caso B) mostra os pontos de cada estabelecimento claramente — o cliente consegue decidir sem dificuldade?
  • ☐ A seleção de tenant precisa de confirmação ("Você selecionou AutoWash Pro — Continuar?") ou é direta (clique → login)?
  • ☐ O prompt de telefone (UC-021 A3) parece intrusivo? Deve ser uma página separada ou um modal/banner na primeira tela pós-login?
  • ☐ Onde o cliente aterrissa após o login bem-sucedido — hotsite logado, /minha-conta, ou de volta ao booking form?
  • ☐ A mensagem de erro "no-tenant" orienta o cliente a acessar o hotsite correto?

Arquitetura

  • shared/login.html → produção: apps/web/app/[tenantSlug]/login/page.tsx (GAP, M124-S02)
  • 01-select-tenant.html → produção: apps/web/app/select-tenant/page.tsx (GAP, M124-S02)
  • 02-phone-completion.html → produção: prompt inline pós-login (GAP, M124-S02)
  • 01b-error.html → produção: apps/web/app/auth/error/page.tsx (GAP, M124-S01)
  • Backend/BFF: todos os endpoints de auth já implementados em M03 ✓
