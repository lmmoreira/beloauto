Screens

Estados alternativos (unhappy paths)

02b Erro — e-mail já cadastrado UC-028 A1 03b Erro — autodesativação bloqueada UC-029 A1 03c Erro — último gerente ativo UC-029 A2

Dry-run checklist

  • Lista mostra Ativo / Convite pendente / Inativo com badges distintos e corretos
  • Sua própria linha (Ana Pereira) não tem botão "Desativar"
  • Botão "+ Convidar membro" → formulário de convite
  • Convidar com e-mail já ativo → erro inline no campo e-mail
  • Convidar válido → volta para lista com toast de confirmação
  • Linha "Convite pendente" tem ação "Reenviar convite" em vez de "Desativar"
  • Botão "Desativar" em outro membro → página de confirmação
  • Confirmar desativação → lista com membro marcado como Inativo
  • Cancelar desativação → volta para a lista
  • Tentativa de autodesativação → erro 403 visível
  • Tentativa de desativar o último gerente ativo → erro 409 visível

Open questions

  • Route location: /dashboard/team/ ou /[slug]/dashboard/team/?
  • Lista mostra membros inativos por padrão, ou só com filtro explícito?
  • Reenviar convite: reabre o formulário de convite, ou uma ação de um clique?
  • Distinção "Convite pendente" vs "Inativo" depende de googleOAuthId / deactivatedBy — não existe um status dedicado no banco. Confirmar se a API expõe esse cálculo ou se o frontend precisa derivá-lo.
