STAFF — Horários

Protótipo do calendário de agendamentos aprovados e gestão de bloqueios/aberturas (UC-010a–d)

O que este protótipo valida

  1. A visualização de semana (week strip) + grade de horários diária é suficiente para o staff ver o dia de trabalho?
  2. A distinção visual entre agendamento aprovado (azul) e bloqueio (cinza hachurado) está clara?
  3. O fluxo FAB → bottom sheet → confirmação para criar bloqueio é adequado ou é necessário uma tela separada?
  4. O aviso "agendamentos existem nesse período" (UC-010a A4) deve bloquear a criação ou apenas informar?
  5. A gestão de dia fechado (domingo) → "Abrir dia especial" está bem posicionada no calendário?
  6. Um dia com abertura especial vs. bloqueio precisam de dois estilos visuais distintos ou um é suficiente?

Fluxo principal — dia aberto

  • 00 Calendário semanal — dia aberto (Ter 16) UC-010 GAP
  • 00 Calendário semanal — próxima semana (Seg 22) UC-010 GAP
  • 01 Criar bloqueio — formulário (FAB → bottom sheet) UC-010a GAP
  • 02 Remover bloqueio — confirmação UC-010b GAP

Fluxo secundário — dia fechado

  • 03 Calendário — dia fechado (Dom 22) UC-010c GAP
  • 04 Abrir dia especial — formulário UC-010c GAP
  • 05 Abertura criada — domingo com janela 09:00–14:00 UC-010c GAP
  • 06 Remover abertura — confirmação UC-010d GAP

Estados alternativos (unhappy paths)

  • 01b Criar bloqueio — 409 Conflict (período já bloqueado) UC-010a A2/A3 ⚠ GAP
  • 01c Bloqueio criado — aviso de agendamentos no período (não bloqueia a criação) UC-010a A4 ⚠ GAP
